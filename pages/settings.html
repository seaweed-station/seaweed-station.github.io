Automatic Download Schedule

Managed by GitHub Actions -- runs in the cloud every hour
🔁 Current schedule: every hour
GitHub Actions runs download_data_supabase.ps1 automatically every hour (00:00, 01:00, 02:00 … UTC), commits any new data to the repo, and deploys it to GitHub Pages. Your PC does not need to be on.
Manual runs
Browser-triggered workflow dispatch has been removed from the public dashboard.
To run the download manually, use the GitHub Actions workflow page in the organization repo.
⚡ Open Actions 🔗 View run history ✎ Edit schedule (change cron)
Changing the frequency
Click Edit schedule above to open the workflow YAML on GitHub. Change the cron line:
cron: '0 * * * *' Common alternatives: '0 */6 * * *' (every 6 h)  |  '0 8 * * *' (daily at 08:00 UTC)

Access Roles

Control who sees which stations and features. Pushed to Supabase so every browser sees the same rules.
How it works
Each role has a password and a set of allowed stations/features.
Stored passwords are no longer shown in the browser UI.
Enter a new password only when rotating a role, and leave it blank to keep the current stored password.
This is still a convenience gate, not a security boundary.

Local Development

Run download scripts manually on your own machine
Optional JSON export for local script runs
Only needed when running download_data_supabase.ps1 manually on your PC. Save your settings first, then export config.json and place it next to the script.
Run a manual download any time
Open PowerShell in the repo folder and run: .\download_data_supabase.ps1 Or with override options: .\download_data_supabase.ps1 -ScheduleType hourly -MaxResults 8000
